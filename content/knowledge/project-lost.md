Mark Project as Lost

Overview:
When a project is no longer moving forward, you can mark it as Lost. This removes it from the active pipeline and records the reason for losing the deal.

How to Access:
On the Project record page, click the "Lost" quick action button. On the Projects Board, click the Lost action icon on the project card.

How to Mark a Project as Lost:
1. Open the Project or find it on the Projects Board
2. Click the "Lost" action
3. Select a reason from the dropdown (e.g., Cancelled, Price Too High, Went with Competitor, No Response)
4. Add optional notes explaining the situation
5. Click Confirm

What Happens:
- The Project sales stage changes to Lost
- The project is removed from the active Projects Board columns
- The lost reason and date are recorded for reporting
- The project can still be accessed through search or filtered list views

Reopening a Lost Project:
If a project comes back to life:
1. Open the Lost project record
2. Use the Reopen action to move it back to an active stage
3. The project reappears on the Projects Board

Common Issues:

Cannot mark as lost:
- Check that the project is in an eligible stage
- Some stages may require specific permissions to mark as lost

Tips:
- Always select the correct lost reason for accurate reporting
- Add notes about why the deal was lost to help improve future sales
- Review lost projects periodically to identify patterns (pricing, competition, timing)
